Tratic
=====================================

차트 기반 매매일지 분석 앱

### 배경

매매 일지를 작성하는게 좋다는 것은 누구나 아는데 왜 작성하지 않을까?  
매매 일지를 작성하는건 귀찮고, 매매 일지 작성의 효과가 눈에 드러나지 않기 때문에  

### 앱의 목표

기술 지표 분석을 통한 매매 기록의 즉각적, 정량적 평가  

### 흐름

1. 유저 로그인
2. 매매기록 입력
3. 차트 호출
4. 지표 생성
5. 매매기록 분석

### 구현

차트를 일일이 저장하지 않고, 요청 들어오면 그 지점부터 과거로 분석에 쓸만큼 캔들을 가져온다

#### 사용자에게 받는 입력
- 마켓
- 심볼
- 시간
- 진입가
- 익절라인
- 손절라인
- 사용한 시간봉

#### 사용자가 받는 출력
리포트 형식의 거래 판단

#### 보장하는 해상도
일봉까지. 단, 일봉 아래 존재하는 해상도는 사용자가 직접 확인해야함

#### 레이트리밋
마켓 API 레이트 리밋 또는 연속된 내부 서버 에러 발생 시 요청을 일정시간 잠굴 것
